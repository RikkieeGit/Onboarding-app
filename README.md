# 🔐 Onboarding API

Backend for the onboarding flow on [paperwork.framer.ai](https://paperwork.framer.ai/) — handles user registration, authentication, and OTP verification via email.

---

## How I Built It

### Frontend — Framer
Custom React component built in Framer with sign in, sign up, OTP verification, and forgot password screens. Google-style minimal dark UI with floating label inputs, responsive for desktop and mobile.

### Backend — Python (FastAPI)
REST API handling user registration and authentication. Users are stored in **Samba Active Directory** running on the same Oracle Cloud server. OTP codes are generated by the API itself and delivered via **Gmail SMTP** — no third party OTP service needed.

### Infrastructure
Deployed on an Oracle Cloud free-tier Ubuntu instance with a free **DuckDNS** domain and **Let's Encrypt** SSL certificate. Runs as a **systemd** service so it starts automatically on boot and restarts on failure.

## Architecture

```
Framer Frontend
      ↓ HTTPS
FastAPI on Oracle Cloud (port 8443)
      ↓              ↓
Samba AD         Gmail SMTP
(store users)    (send OTP)
```

---

## API Endpoints

| Method | Endpoint | What it does |
|---|---|---|
| `POST` | `/register` | Validate email, generate OTP, send to email |
| `POST` | `/signin` | Check AD credentials, send OTP to email |
| `POST` | `/verify` | Verify OTP, create AD user if signup flow |
| `POST` | `/forgot-password` | Send reset OTP to email |
| `GET` | `/health` | Health check |

## Run Locally

```bash
# Clone the repo
git clone https://github.com/RikkieeGit/Onboarding-app.git
cd Onboarding-app

# Copy env and fill in your values
cp .env.example .env

# Install dependencies
pip3 install -r requirements.txt

# Run
uvicorn main:app --host 0.0.0.0 --port 8443
```
